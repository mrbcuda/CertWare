Context Nodes

Background

A context node provides background information for a goal or solution node. The background information captures constraints on the applicability of the node to which it is attached. Context provides information necessary for an Argument (or other GSN element) to be understood or amplified. Context may include a statement which limits the scope of an Argument in some way.

Application

To add a context node to the case argument model, right-click on a Goal or Solution node, choose Add Child... from the pop-up menu, and choose Context. Select the new context node and then complete its properties in the property sheet form.
