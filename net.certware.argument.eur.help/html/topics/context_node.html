Context Nodes

Background

Context provides the inputs or background information that a goal or other goal structuring element requires for it to be understood or satisfied. It will include analysis results, hazard logs, etc. in some senses models and assumptions could be regarded as special cases of context, but are treated as separate entities because of their importance in defining goal structures.

Application

To add a context node to the case argument model, right-click on an Argument or Solution node, choose Add Child... from the pop-up menu, and choose Context. Select the new context node and then complete its properties in the property sheet form.
